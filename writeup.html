Chat App Writeup (What, Where, and Who)

Action 1: Create a Public Thread

What?: Create a new thread with a specific title (the topic). This is represented as a Graffiti Object with the following value: 

        
        {
          "activity": "Create",
          "type": "Thread",
          "title": title, // chosen by user
          "classId": classId, // based on MIT course number (e.g. 6.4500)
          "channel": crypto.randomUUID(),
          "published": Date.now()
        }

Where? New threads should be discoverable by anyone in the class so we use the hardcoded name associated with the class (e.g. "mit:class:6.4500") so people can query it and discover new threads.

Who? Anyone in the class can see all the messages in a public thread, so there is no allowed list.

Action 2: Create a Private Thread

What?: Create a new thread and share it with specific individuals. This is represented as a Graffiti Object with the following value: 

        
        {
          "activity": "Create",
          "type": "Thread",
          "channel": crypto.randomUUID(),
          "published": Date.now()
        }

Where? Private threads should only be discoverable by the users that have access to it, so it appears in each allowed users' inbox.

Who? Allowed list contains the members' identifiers.

Action 3: Send a Message

What?: Send a message into a specific thread. This is represented as a Graffiti Object with the following value: 

        
        {
          "content": content, // chosen by user, 
          "published": Date.now() 
        }

Where? Messages should exist in the thread they are sent in. This channel is created with crypto.randomUUID() and persisted/shared in the the "Create Public/Private Thread" object.

Who? For private threads, only users in the allowed list. For public threads no allowed list.
